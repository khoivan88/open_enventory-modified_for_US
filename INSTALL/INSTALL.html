Installation of the server components

  • Install Xampp with Apache, MySQL, phpMyAdmin and PHP5+. Currently, version 7.3.7 is recommended to avoid problems introduced by MariaDB 10.4+. If asked to set a MySQL root password, please do not forget it, as it is very important for the completion of the installation.
  • Log into phpMyAdmin and set a password for the root users.
  • For the PDF preview, ghostscript needs to be installed. The constant ghostscript_command in the file "lib_global_settings.php" must point to the absolute path (double backslashes) of the gswin32c.exe, under Linux is gs already set.
  • For full text search in literature PDFs, pdftotext.exe from the Xpdf or Poppler package is recommended, if it is not installed, set the constant pdftotext_command in the file "lib_global_settings.php" to "" for using Ghostscript to perform this task, which is less accurate.
  • For BMP, TIF, PCX, RLE, TGA, WMF support, ImageMagick needs to be installed - activate the option "Install legacy utilities" during installation if using version 7 or above. The constant imagemagick_command in the file "lib_global_settings.php" must point to the absolute path (double backslashes) of the convert.exe, under Linux is convert already set.
  • For the DOC/XLS/... preview (optional), it is also required to install a Java Runtime Environment and Libreoffice.org. The constant java_command must point to the absolute path (double backslashes) of the java.exe. At system startup, the command 
    [path]\soffice.exe --accept="socket,host=127.0.0.1,port=8100;urp;" --invisible --nologo --nofirststartwizard --norestore
    should be called.
  • the following extensions are required, but for current versions of Xampp, they are already activated by default:
    • ftp (for the transfer of spectra),
    • gd (for structures and preview of spectra),
    • imagemagick (php_imagick.dll, thanks to Marek Štítek for this hint)
    • json (for the asynchronous communication),
    • mbstring,
    • mysqli and
    • session.
    Under Xampp/Windows, these extensions are installed by default and must be activated in the php.ini (make sure the you edit the right php.ini, sometimes there are multiple present).
  • In the php.ini, you must increase the POST limits of Suhosin (if available) significantly, i.e.: 
    suhosin.post.max_value_length=128000
suhosin.post.max_vars=2000
suhosin.request.max_value_length=128000
suhosin.request.max_vars=2000
    When working with very large molecules, even higher values may be required.
  • Unpack the source code into xampp\htdocs, perhaps into a subfolder
  • Open localhost/[installation directory]/index.php in your browser and log on using the username root and the respective password (empty passwords are not allowed and will not even be checked, i.e. a root password must be set). Databases which do not exist will be created. Users can be created at "Settings > Users" The credentials at the login screen will be passed to the MySQL-Server, i.e. user accounts are database users with the respective permissions.
  • If the program runs on a Windows7 server (Win8 or Vista might also be affected) very slow, it may help to activate/add the following line to c:\windows\system32\drivers\etc\hosts: 
    127.0.0.1 localhost
  • Install in SynapticYumZYpper the following packages:
    • php-mysql (many of the following packages will be installed automatically as dependencies), php, php-cli, php-common, php-gd, phpmyadmin, php-pear, mysql-server, mysql-client, mysql-common, apache2 php7-pear, apache2, php7, php7-ftp, php7-mysql, apache2-mod_php7, mariadb mariadb-tools, phpMyAdmin (according to this guide) php-mysql (many of the following packages will be installed automatically as dependencies), php, php-gd, php-mbstring, php-pear, httpd, zlib-devel, httpd.x86_64
    • optionally ghostscript (Preview of the 1st page of PDFs within the browser as PNG), imagemagick (BMP, TIF, PCX, RLE, TGA, WMF support) and libreoffice.org,oracle-java-... via java-package (Preview of the 1st page of DOCs, XLSs,... within the browser as PNG)
    • on Linux Mint 19.3, php and libapache2-mod-php must be installed and enabled via 
      sudo a2dismod mpm_event
sudo a2enmod php7.2
sudo service apache2 restart
      - otherwise only the PHP source code is displayed
  • Log into MariaDB via 
    sudo mysql -u root
    and set a MySQL root password by typing 
    SET PASSWORD = PASSWORD('my secret pw');
UPDATE mysql.user SET plugin = '';
FLUSH PRIVILEGES;
    - or log into phpMyAdmin and set a password for the root users.
  • For the DOC preview (optional), add 
    bash -l -c "soffice --accept=\"socket,host=127.0.0.1,port=8100;urp;\" --invisible --nofirststartwizard --norestore --headless &"
    before 
    exit 0
    in /etc/rc.local.
  • Unless you know in detail how to tune the configuration, we recommend to set 
    SELINUX=permissive
    in /etc/selinux/config.
  • The POST limits of Suhosin in /etc/php5/apache2/conf.d/suhosin.ini should be increased significantly, i.e.: 
    suhosin.post.max_value_length=128000
suhosin.post.max_vars=2000
suhosin.request.max_value_length=128000
suhosin.request.max_vars=2000
    When working with very large molecules, even higher values may be required.

Installation der Serverkomponenten

  • Installieren Sie Xampp mit Apache, MySQL, phpMyAdmin und PHP5+. Gegenwärtig ist Version 7.3.7 zu empfehlen, um Probleme mit MariaDB 10.4+ zu vermeiden. Falls Sie aufgefordert werden, ein MySQL-Root-Paßwort zu vergeben, dann vergessen Sie es keinesfalls, weil es für die spätere Installation benötigt wird.
  • Loggen Sie sich in phpMyAdmin ein und vergeben Sie ein Paßwort für die root-Benutzer.
  • Für die PDF-Vorschau muß Ghostscript installiert werden und in der Datei "lib_global_settings.php" die Konstante ghostscript_command den absoluten Pfad (Backslashes verdoppeln) zur gswin32c.exe enthalten.
  • Für Volltextsuche in Literatur-PDFs wird die Installation von pdftotext.exe aus dem Xpdf- oder Poppler-Utils-Paket empfohlen. Falls es nicht installiert ist, sollte die Konstante pdftotext_command in der Datei "lib_global_settings.php" auf "" gesetzt werden. In diesem Fall wird für diese Aufgabe Ghostscript verwendet, das weniger akkurate Ergebnisse liefert.
  • Für BMP, TIF, PCX, RLE, TGA, WMF-Support muß ImageMagick installiert werden - ab Version 7 mit der Option "Install legacy utilities". Die Konstante imagemagick_command in der Datei "lib_global_settings.php" den absoluten Pfad (Backslashes verdoppeln) zur convert.exe enthalten.
  • Für die DOC/XLS/...-Vorschau (optional) muß außerdem eine Java Runtime Environment und Libreoffice.org installiert werden und die Konstante java_command den absoluten Pfad (Backslashes verdoppeln) zur java.exe enthalten. Beim Systemstart

Completion of the installation

  • Make sure that the Apache and MySQL services are running in the Xampp control panelby starting the services apache2httpd and mysqlmariadb via 
    sudo service [Name] start
  • Unpack the source code into c:\xampp\htdocs\/var/www/html//srv/www/[installation directory] and set the permissions to 755 to make them accesible by Apache.
  • Open localhost/[installation directory]/index.php in your browser and log on using the username root and the respective password (empty passwords are not allowed and will not even be checked, i.e. a root password must be set). Databases which do not exist will be created. Users can be created at "Settings > Users"
The credentials at the login screen will be passed to the MySQL-Server, i.e. user accounts are database users with the respective permissions.

Abschluß der Installation

  • Stellen Sie sicher, daß der Apache- und MySQL-Dienst laufen, über das Xampp-Control-Panelindem Sie die Dienste apache2httpd und mysqlmariadb über 
    sudo service [Name] start
    starten    .
  • Den Quellcode in c:\xampp\htdocs\/var/www/html//srv/www/[Installationsverzeichnis] entpacken und die Berechtigungen auf 755 setzen, damit der Apache die Dateien lesen kann.
  • Rufen Sie im Browser localhost/[Installationsverzeichnis]/index.php auf und loggen Sie sich mit dem Benutzer root und dem entsprechenden Paßwort (leere Kennwörter sind nicht zulässig, d.h. es muß ein root-Paßwort gesetzt sein) ein, nicht vorhandene Datenbanken werden erstellt. Dann können Sie bei "Einstellungen > Benutzer" diese anlegen.
Die Authentifizierung wird beim Login an den MySQL-Server weitergereicht, d.h. Benutzerkonten sind Datenbankbenutzer mit entsprechenden Berechtigungen.
